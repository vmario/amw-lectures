POSIX

  • Portable Operating System Interface Wyspecyfikowany przez IEEE Standaryzuje API, powłoki i narzędzia systemowe Rozszerza bibliotekę standardową C – C POSIX library

Systemy zgodne z POSIX-em

    Certyfikowane: HP-UX, IBM AIX, Solaris, OS X, *QNX Neutrino* Zgodne: illumos (dawniej OpenSolaris), rodzina BSD, Linux (w tym Android), *Nucleus RTOS, NuttX, RTEMS, VxWorks* Protezy: Cygwin dla Windows

Busybox

Programy wywoływane są przez linki symboliczne, np.:␋$ ls –l /bin␋… cp -> busybox␋… ls -> busybox␋… mv -> busybox␋

MMU

Tłumaczy adresy wirtualne na fizyczne za pomocą PTE (Page Table Entry) i jego cache TLB (Translation Lookaside Buffer) Dzieli pamięć na segmenty (na ogół rzędu 4KiB), które podlegają ochronie

Kernel

Ma pełny dostęp do pamięci fizycznej i zarządza pamięcią wirtualną Ma pełny dostęp do zasobów sprzętowych Ma pełny dostęp do przerwań sprzętowych Zarządza pracą programów przestrzeni użytkownika m.in. przez scheduler Udostępnia sterowniki urządzeń, systemy plików, stosy protokołów sieciowych itp.

Programy w przestrzeni użytkownika

Mają ograniczony dostęp do pamięci, zwłaszcza pamięci innych programów Na ogół nie wykorzystują bezpośredniego dostępu do sprzętu Podlegają schedulerowi Mogą na nie zostać nałożone ograniczenia (vide: ulimit) np. odnośnie czasu procesora, zużycia pamięci, liczby otwartych plików itp.

U-Boot

Wstępna konfiguracja mikrokontrolera Inicjalizacja interfejsu Ethernet, karty pamięci Uruchomienie terminala RS-232 Ładowanie systemu operacyjnego z pamięci nieulotnej, USB, via Ethernet (TFTP) Zarządzanie skryptami i zmiennymi środowiskowymi Wyświetlenie splashscreena Obsługa trybów awaryjnych i serwisowych Diagnostyka urządzenia, flashowanie pamięci, testowanie TCP/IP itp.
